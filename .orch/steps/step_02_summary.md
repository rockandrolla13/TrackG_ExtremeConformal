Implemented `src/data/download_fx.py` as a production-oriented HistData downloader for `EURUSD`, `GBPUSD`, and `USDJPY`, covering 2010-01-01 through 2024-12-31 at 1-minute OHLCV frequency. The script fetches HistData's optional `tk` token, downloads monthly ZIP archives with retry handling, extracts CSV payloads automatically, normalizes them to the required `datetime,open,high,low,close,volume` schema, and appends them into `data/fx/raw/{PAIR}_1min_2010_2024.csv`. It logs request progress, ZIP/CSV/output sizes, and estimated remaining time, and it raises on unrecoverable failures so the CLI exits with status code 1.

Because `.orch/steps/step_01_summary.md` only reported failed web research, the implementation used the repository's existing sourcing guidance in `docs/data_sourcing_report.md`, which identifies HistData.com as the top recommended free FX source. Resume behavior is based on the last timestamp already present in each aggregate CSV: completed pairs are skipped, interrupted runs resume from the current month and filter out previously written rows, and a 2.5-second inter-request delay is enforced. Unit tests for the helper functions were updated in `tests/test_download_fx.py`, but full pytest execution could not be completed in this environment because `pandas` is not installed.
