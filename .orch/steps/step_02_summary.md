Implemented `src/data/download_fx.py` as a Dukascopy-first FX downloader for `EURUSD`, `GBPUSD`, and `USDJPY`, targeting `2010-01-01` through `2024-12-31` and writing aggregate outputs to `data/fx/raw/{PAIR}_1min_2010_2024.csv`. The script now uses Dukascopy's hourly `datafeed.dukascopy.com` `.bi5` tick archives, decompresses and parses the binary records, converts them into minute-level `datetime,open,high,low,close,volume` bars, and appends them with resume-aware behavior based on the last written timestamp. It includes progress logging, file-size logging, request retries, 2.5-second throttling between requests, explicit failure propagation, and a `main()` entrypoint that exits with status code `1` on unrecoverable download errors.

Added a TrueFX fallback path that makes a best-effort attempt to discover monthly ZIP downloads from the official historical downloads page, parse fallback tick CSVs, and aggregate them to the same 1-minute schema if Dukascopy is inaccessible. Updated `tests/test_download_fx.py` to cover the new Dukascopy URL construction, `.bi5` parsing, tick-to-bar aggregation, TrueFX parsing, ZIP extraction, and resume helpers. `pytest tests/test_download_fx.py` could not run in the current environment because `pandas` is not installed locally, but `python -m py_compile src/data/download_fx.py tests/test_download_fx.py` completed successfully.
