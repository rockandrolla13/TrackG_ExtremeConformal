Implemented [src/data/download_crypto.py](/media/ak/10E1026C4FA6006E/GitRepos/TrackG_ExtremeConformal/src/data/download_crypto.py) to mirror the structure and operational behavior of the FX downloader while targeting Binance Data Vision monthly 1-minute spot klines for `BTCUSDT` and `ETHUSDT`. The script writes consolidated CSVs into `data/crypto/raw/` using the required `{PAIR}_1min_2018_2024.csv` naming, normalizes output to `datetime,open,high,low,close,volume`, supports resume from the last written timestamp, skips already complete files, logs request progress with sizes and ETA, retries transient failures up to 3 times, and raises on unrecoverable errors so `python src/data/download_crypto.py` exits non-zero.

Updated [tests/test_download_crypto.py](/media/ak/10E1026C4FA6006E/GitRepos/TrackG_ExtremeConformal/tests/test_download_crypto.py) to cover the same helper-level behavior as the FX test suite: month iteration, URL construction, CSV parsing, ZIP extraction, byte formatting, last-timestamp resume support, and end-of-range completion checks. Verification in this environment was limited to static execution checks because the local environment may not have all runtime dependencies installed for `pytest`.
