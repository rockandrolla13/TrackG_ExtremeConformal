Implemented the Binance Data Vision crypto downloader in [src/data/download_crypto.py](/media/ak/10E1026C4FA6006E/GitRepos/TrackG_ExtremeConformal/src/data/download_crypto.py) using the same structure as the FX script: monthly iteration from January 2018 through December 2024, retrying HTTP downloads, ZIP validation/extraction, resume-from-existing-output behavior based on the last written `datetime`, progress logging with byte sizes and ETA, and normalized output to `data/crypto/raw/{PAIR}_1min_2018_2024.csv` for `BTCUSDT` and `ETHUSDT`. The script reads Binance spot 1-minute kline ZIP archives from `https://data.binance.vision/data/spot/monthly/klines/{SYMBOL}/1m/{SYMBOL}-1m-YYYY-MM.zip`, converts millisecond `open_time` values to naive datetime strings, keeps only `datetime,open,high,low,close,volume`, and now exits with status code `1` on any unrecoverable failure via `SystemExit(1)` in `main()`.
